# Homepage Lazy Quarter Loading Design

**Date:** 2026-04-17

## Goal

把主站首页的首屏加载从“下载全量历史数据并补齐所有公司缓存”改为“仅下载公司索引与各公司最新季度所需数据”，让页面能快速显示公司列表和默认图表；只有当用户查看非最新季度时，才按公司懒加载完整历史数据。

## Current Problem

- 首页启动会请求 `data/earnings-dataset.json`，当前体积约 18 MB。
- 启动后还会为每家公司继续请求 `data/cache/<company>.json` 做补全，形成额外串联请求。
- 首屏只需要公司列表和默认图，但当前数据管线为全部公司、全部季度买单。

## Proposed Design

### 1. 新增轻量首页索引

构建阶段新增一个公开文件，例如 `data/dataset-index.json`，包含：

- 全局元信息：`generatedAt`、`companyCount`
- 公司轻量索引：`id`、`ticker`、`nameZh`、`nameEn`、`slug`、`rank`、`isAdr`、`brand`
- 公司季度元信息：`quarters`、`coverage`
- 默认展示所需的最新季度图表数据：
  - `latestQuarter`
  - `financials[latestQuarter]`
  - 与最新季度渲染直接相关的最小字段，如 `statementPresets[latestQuarter]`
  - 若最新季度图表依赖结构历史，保留最新季度对应的最小结构数据，而不是整段历史

### 2. 前端改为两阶段加载

- `boot()` 首先加载 `dataset-index.json`
- 页面初始化、公司列表、默认公司选择、最新季度渲染，全部基于这个轻量索引完成
- 不在启动阶段调用“为所有公司补齐 fallback”的全量逻辑

### 3. 按公司懒加载完整历史

- 新增“确保公司历史已加载”的前端方法
- 当用户选择一个公司的非最新季度，或该图表必须依赖完整历史时，再请求 `data/cache/<company>.json`
- 请求成功后把该公司的完整 payload 合并进内存缓存，并标记为已加载，避免重复请求

### 4. 渲染兼容策略

- 最新季度图默认可直接渲染
- 30 季柱状图、历史桑基图仍然依赖完整公司缓存，但只在用户进入相应季度时才下载
- 台积电、ASML 等特殊修正仍以公司缓存为最终真源；首页索引只是首屏快照

## Error Handling

- 如果轻量索引加载失败，维持当前“数据加载失败”提示
- 如果某公司历史懒加载失败：
  - 最新季度仍可看
  - 非最新季度切换时给出明确状态提示，而不是整页卡住

## Testing Strategy

- 构建脚本测试：验证首页索引仅包含最新季度最小字段
- 前端单元测试：验证选择最新季度不触发公司历史请求，切到历史季度才触发
- 回归测试：确认台积电最新季度费用拆分仍为“研发 / 销售及管理费用”
